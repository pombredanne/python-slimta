slimta.smtp.auth.standard¶

Module containing the built-in supported authentication mechanisms.

class slimta.smtp.auth.standard.Plain(verify_secret, get_secret)¶

Bases: slimta.smtp.auth.ServerMechanism, slimta.smtp.auth.ClientMechanism

PLAIN authentication mechanism. This is the primary mechanism to use on encrypted channels.

See RFC 4616 for details.

name = 'PLAIN'¶

This mechanism identifies itself as PLAIN.

secure = False¶

This mechanism is not secure for use on unencrypted channels.

class slimta.smtp.auth.standard.Login(verify_secret, get_secret)¶

Bases: slimta.smtp.auth.ServerMechanism, slimta.smtp.auth.ClientMechanism

LOGIN authentication mechanism. Simply a back-and-forth request from the client for its username and password, base64-encoded.

name = 'LOGIN'¶

This mechanism identifies itself as LOGIN.

secure = False¶

This mechanism is not secure for use on unencrypted channels.

class slimta.smtp.auth.standard.CramMd5(verify_secret, get_secret)¶

Bases: slimta.smtp.auth.ServerMechanism, slimta.smtp.auth.ClientMechanism

CRAM-MD5 authentication mechanism. With this mechanism, the server presents an arbitrary challenge string and the client must MD5 hash that string using their password as a key. This means the password is not ever communicated in a reversible form. However, it also means the server must have access to un-hashed passwords, which is not always possible or desired.

See RFC 2195 for details.

name = 'CRAM-MD5'¶

This mechanism identifies itself as CRAM-MD5.

secure = True¶

This mechanism is secure for use on unencrypted channels.

hostname = 'mail.slimta.org'¶

This is the hostname used when generating the initial challenge.

requires_get_secret = True¶

This mechanism requires direct access to the original password using get_secret(). Auth sub-classes must implement that method or this mechanism will be unavailable.
